## Vision Modes

Select via **Settings → AI Vision Mode**. Applies at the next session start.

### 🔬 Simulation Mode (default)
- No camera required, no API key required
- Synthetic detections generated by `SimulationVisionProvider`
- Identical guardian/alert pipeline to live modes
- Use for development, testing, and demo

### ⚡ Live Local AI Mode (recommended)
- **Live camera required** (browser `getUserMedia`)
- **TensorFlow.js COCO-SSD** runs entirely on-device
- 80 detectable object classes (people, vehicles, furniture, traffic signs, etc.)
- Typical detection latency: 50–200 ms per frame on a modern device
- No API key needed — works offline
- Falls back to simulation if camera is denied

### ☁️ Live Cloud AI Mode
- **Live camera required + API key required**
- Camera frames sent to Gemini or OpenAI Vision via `/api/vision/analyze-frame`
- Richer scene understanding and natural language descriptions
- Set `GEMINI_API_KEY` or `OPENAI_API_KEY` in Replit Secrets
- Configured via `VISION_PROVIDER` environment variable (`gemini` | `openai`)
- Falls back to simulation if API key is missing

## OCR — Reading Signs and Labels

Enable via **Settings → OCR (Read Signs & Labels)**.

- Uses **Tesseract.js v5** (WebAssembly, runs on-device)
- Processes camera frames approximately every 15 seconds
- Reads English text by default
- Results are announced via TTS if text is detected
- Hazard keywords (STOP, DANGER, WARNING, etc.) trigger priority alerts

---

## Text-to-Speech

Uses the **Web Speech API** (`window.speechSynthesis`) — a real browser capability.

- Configure rate, volume, and voice in **Settings → Voice Output**
- Priority queue: critical → high → normal → low
- Critical alerts interrupt any ongoing speech
- Works in Chrome, Edge, Firefox, Safari

---

## Speech Recognition

Uses the **Web Speech API** (`SpeechRecognition` / `webkitSpeechRecognition`) — real browser STT.

- Full support in Chrome and Edge
- Limited/no support in Firefox and Safari
- Commands parsed by `voiceCommandEngine.ts`
- 15+ command intents (start, stop, describe, navigate, emergency stop, etc.)
- Microphone indicator shown in the session page

## Latency Measurement

Latency is measured automatically during live sessions.

**Metric:** Capture-to-decision pipeline time (frame start → safety decision complete).

| Mode | Typical Latency |
|------|----------------|
| Simulation | < 200 ms (local API) |
| Local AI (COCO-SSD) | 200–600 ms |
| Cloud AI (Gemini/OpenAI) | 800–2500 ms |

Latency is displayed live in the Session page camera panel.

**What it measures:**
- Frame capture / detection time
- Scene description API call
- Hazard evaluation
- Safety decision
- _Does not_ include TTS speech duration

## Architecture

```
Browser (client-side)
├── useCameraCapture.ts      — getUserMedia → live video element
├── useLocalObjectDetection.ts — TF.js COCO-SSD → Detection[]
├── useOCR.ts                — Tesseract.js → OCR text
├── useVisionSession.ts      — orchestrates frame loop
│   ├── Local AI mode: inject TFJS detections → skip server vision call
│   ├── Simulation/Cloud: POST /api/vision/analyze-frame
│   └── Latency tracking: captureStart → speak()
├── useAudioGuidance.ts      — Web Speech API TTS (real)
└── useVoiceCommands.ts      — Web Speech API STT (real)

Server (Express proxy + Next.js API routes)
├── /api/vision/analyze-frame — SimulationVisionProvider | GeminiVisionProvider | OpenAIVisionProvider
├── /api/hazards/evaluate    — hazard classification from detections
├── /api/safety/decide       — safety decision from alerts
└── /api/scene/describe      — scene summary from detections
```

## Configuration

All settings are persisted to `localStorage` via `settingsStore.ts`.

| Setting | Key | Values |
|---------|-----|--------|
| Vision Mode | `visionMode` | `simulation` \| `local-ai` \| `cloud-ai` |
| OCR Enabled | `ocrEnabled` | `true` \| `false` |
| Voice Rate | `speechRate` | 0.5–2.0 |
| Voice Volume | `speechVolume` | 0–1 |
| Voice Name | `voiceName` | Browser voice name |

Server-side provider selection uses the `VISION_PROVIDER` environment variable (`simulation` | `gemini` | `openai`). If unset, defaults to `simulation`.
