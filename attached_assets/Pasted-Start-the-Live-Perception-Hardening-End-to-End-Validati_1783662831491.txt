──────────────────────────────

──────────────────────────────
3. SMART ALERT COORDINATION
──────────────────────────────

Ensure OCR, vision, navigation, and companion messages do not speak over each other.

Required priority:

1. Critical hazard
2. High-risk navigation warning
3. Emergency/system failure
4. Important OCR
5. Normal navigation
6. Scene description
7. Companion information

Critical alerts must:
- interrupt current speech
- clear lower-priority queued speech
- trigger haptic fallback when available

Do not repeatedly speak the same warning unless:
- risk increases
- distance decreases materially
- direction changes
- the warning cooldown expires

──────────────────────────────
4. REAL FAILURE RECOVERY
──────────────────────────────

Test and implement graceful recovery for:

- camera permission denied
- camera disconnected
- local model load failure
- cloud provider timeout
- no internet
- weak internet
- OCR worker crash
- speech recognition unavailable
- TTS unavailable
- navigation location unavailable
- battery-critical mode
- overheating mode

Core offline safety must continue without cloud access.

Never fail silently.

──────────────────────────────
6. USER-FACING WORDING
──────────────────────────────

Remove technical or robotic speech from normal operation.

Bad:
“Object detection confidence 0.86.”

Good:
“Obstacle ahead, slightly to your right.”

Bad:
“Provider failure, switching mode.”

Good:
“Online vision is unavailable. Offline safety mode is active.”

Never say:
“Safe to cross.”

Use:
“The crossing appears clear, but please check carefully.”
