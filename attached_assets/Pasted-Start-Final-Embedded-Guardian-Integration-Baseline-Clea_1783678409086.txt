3. CONNECT ALERT QUALITY AND COORDINATION
==================================================

Embedded runtime speech must use the same rules as the live browser system.

Required priority:

1. Critical hazard
2. High-risk navigation warning
3. Hardware/system failure
4. Important OCR
5. Normal navigation
6. Scene description
7. Companion information

Implement:

- duplicate suppression
- severity cooldowns
- risk-increase bypass
- distance-decrease bypass
- speech interruption
- clearing lower-priority queued speech
- haptic fallback

Critical alerts must never wait behind normal narration.

==================================================

==================================================
5. CONNECT AUDIO AND HAPTIC ADAPTERS
==================================================

Guardian output must reach the active embedded adapters.

Audio requirements:

- play speech
- interrupt current speech
- stop noncritical speech
- repeat last guidance
- adjust volume
- report playback failure

Haptic requirements:

- stop immediately
- hazard left
- hazard right
- obstacle ahead
- hardware failure
- low battery

If audio fails:

- use haptic fallback;
- announce through restored audio when available;
- never silently discard critical warnings.
