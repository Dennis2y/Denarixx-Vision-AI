Start the Embedded Runtime Reality Check & Stub Elimination Program.

This is NOT a new roadmap phase.

Do not add more conceptual engines.
Do not claim that physical hardware works inside Replit.
Do not describe mocked values as real measurements.

GOAL

Turn the current hardware-bring-up architecture into an honest, executable embedded software baseline.

The program must:

1. stop embedded-prototype mode from silently using simulation adapters;
2. add a genuine local ONNX inference implementation;
3. create executable Linux adapter boundaries;
4. keep mocks only for tests;
5. correct all misleading documentation.

==================================================
1. FIX ADAPTER SELECTION
==================================================

The current startPrototypeRuntime.ts always calls:

assembleSimulationAdapters(true)

even when DENARIXX_HAL_ADAPTER=embedded-prototype.

Replace this with a real adapter factory.

Create or update:

src/runtime/adapters/createHardwareAdapterSet.ts

It must select exactly one mode:

- simulation-test
- browser-development
- embedded-prototype

Required behavior:

simulation-test
→ uses only simulationTestAdapter

browser-development
→ uses browser adapters

embedded-prototype
→ uses only embeddedPrototypeAdapter

Embedded-prototype must never silently fall back to simulation.

If required hardware cannot initialize in embedded-prototype mode:

- fail startup clearly, or
- enter explicit degraded mode with no fabricated detections.

Return a structured result:

- requestedMode
- activeMode
- initializedAdapters
- unavailableAdapters
- startupErrors
- isSimulation

Add a safety assertion:

If requestedMode === embedded-prototype
and activeMode === simulation-test
then throw a startup safety error.

==================================================

The existing localInferenceEngine currently creates mocked classes:

Create:

src/runtime/inference/onnxLocalInferenceProvider.ts

Use ONNX Runtime for Node/Linux where supported.

Preferred package:

onnxruntime-node

If the Replit environment cannot install native ONNX Runtime:

- keep the implementation behind dynamic import;
- provide a clear unsupported-environment result;
- do not substitute fabricated detections;
- retain a mock provider only for automated tests.

The ONNX provider must implement:

- load model from DENARIXX_LOCAL_MODEL_PATH
- verify file exists
- verify model loads
- inspect input and output names
- preprocess RGB frame
- execute inference
- decode outputs
- apply confidence threshold
- run non-maximum suppression where required
- return real measured inference latency
- release model resources on shutdown

Return detections containing:

- className
- confidence
- boundingBox
- source
- capability
- isSimulated=false

Do not mark unsupported classes as reliably detected.
