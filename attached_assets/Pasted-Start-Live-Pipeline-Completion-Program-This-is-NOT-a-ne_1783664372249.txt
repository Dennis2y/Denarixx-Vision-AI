==================================================
1. CONNECT ALERT COORDINATION TO REAL SPEECH
==================================================

Integrate alertCoordinationEngine into useVisionSession.ts.

Every spoken output must go through the coordination queue.

Required priority:

1. Critical hazard
2. High-risk navigation warning
3. System or hardware failure
4. Important OCR
5. Normal navigation
6. Scene description
7. Companion information

Critical alerts must:

- interrupt current speech
- clear lower-priority queued speech
- trigger haptic fallback when enabled
- never be blocked by cooldown

Replace direct speak() calls where appropriate with coordinated dispatch.

==================================================
2. CONNECT ALERT QUALITY ENGINE
==================================================

Integrate alertQualityEngine into the live runFrame pipeline.

The engine must decide:

- whether to speak
- whether to stay silent
- whether the message is a duplicate
- whether risk increased
- whether distance decreased
- whether cooldown expired
- whether wording should be improved

Use guardianWordingEngine for directional, human-friendly guidance.

Examples:

Bad:
“Person detected.”

Good:
“Someone is approaching from your left.”

Bad:
“Obstacle confidence 0.89.”

Good:
“Obstacle ahead, slightly to your right.”

==================================================
3. CONNECT OCR TO THE GUARDIAN
==================================================

Feed OCR results into the Guardian and alert coordination pipeline.

OCR should speak automatically only when relevant.

Priority examples:

High:
- medicine warnings
- emergency signs
- road closure signs
- danger notices

Normal:
- street names
- bus numbers
- menus
- document text

Do not read all visible text continuously.

Add OCR deduplication so the same text is not repeatedly spoken.

==================================================
4. CONNECT NAVIGATION INTELLIGENCE
==================================================

Integrate navigationIntelligenceEngine into useVisionSession.ts.

Per frame or sensor update, combine:

- GPS
- heading
- movement state
- spatial detections
- landmarks
- Guardian risk
- crossing state

Navigation guidance must enter the same speech priority queue.

Examples:

“Continue straight for about ten metres.”

“Turn slightly left.”

“Stop before the crossing.”

“The doorway is ahead on your right.”

Never say:
“Safe to cross.”

Use:
“The crossing appears clear, but please check carefully.”

==================================================
5. CONNECT VOICE INTERACTION ENGINE
==================================================

Use voiceInteractionEngine as the central speech queue controller.

Support:

- speech interruption
- repeat last guidance
- emergency stop
- clearing lower-priority speech
- pause and resume
- provider failure announcements
- offline-mode announcements

Do not allow OCR, navigation, scene description, and companion messages to speak over each other.

==================================================
6. CONNECT NETWORK AND FALLBACK ENGINES
==================================================

Connect:

- networkMonitorEngine
- connectivityFallbackEngine
- failureRecoveryEngine

Announce meaningful changes only once.

Examples:

“Internet is unavailable. Offline safety mode is active.”

“Online enhancement has returned.”

“Camera input is unavailable. Please stop and check carefully.”

Do not repeatedly announce the same state.

==================================================
7. UNIFY THE LIVE PIPELINE
==================================================

Use livePerceptionEngine as the orchestration layer where practical.

Target pipeline:

Camera / glasses sensor input
→ vision provider
→ local detections
→ OCR
→ spatial and navigation context
→ Guardian decision
→ alert quality
→ alert coordination
→ speech or haptic output
→ decision trace
→ latency metrics

If useVisionSession remains the orchestrator, clearly document why and remove or repurpose disconnected duplicate orchestration code.

Do not leave two competing pipeline implementations.
